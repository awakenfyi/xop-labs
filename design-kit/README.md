# design-kit

**The full design work kit, in the Lyra framework — from brief to font system, every phase governed, every verdict labeled.**

Slop is an unnamed default doing the work at distribution scale. This kit makes every default sign its work: a named stance, a recorded brief, craft checks compiled from the stance's own tokens, and taste verdicts that never wear an anchor's uniform. Governing law: [name-the-stance.xop.md](name-the-stance.xop.md). Research foundation: [STORM-REPORT.md](STORM-REPORT.md) (10-agent xop-storm run, adversarially verified).

**Structural blueprint:** professional-grade brand systems, studied privately. The two moves they get right that we adopt wholesale: **principles as the spine** (every layer — type, illustration, motion — expressed *per principle* = a posture library, hand-built) and **min-to-max expression ranges** per artifact (ship a dial, never a point — flattening becomes structurally impossible). Their briefs-as-artifacts and per-section don'ts map to our intent gate and tell-scanners.

Each phase ships three things: the **xOP** (the law), the **skill** (the runtime), and the **prompt pack** (the starters). Phases 2, most of 4, and part of 5 already exist — the kit assembles what a year already built and names what's missing.

## The position

The taste-data model sells rubrics into model weights — closed, verdict-shaped, one curated pool. This kit ships the method in the open, kind-labels every verdict, keeps schools plural by law, and self-applies its own rules (this README wears DESIGNED). **We don't own taste. We make the defaults sign their work.**
